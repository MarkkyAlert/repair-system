# ⚙️ คู่มือผู้ดูแลระบบ (ADMIN-GUIDE)

คู่มือนี้สำหรับ **admin** ที่ตั้งค่าและดูแลระบบผ่านหน้าเว็บ — **ไม่ต้องแก้โค้ด** ทุกอย่างทำผ่านเมนู
เข้าเมนู **"ตั้งค่าระบบ" (`/admin`)** หลังล็อกอินด้วยบัญชี admin จะเห็นแท็บต่าง ๆ ดังนี้

> 🧭 ระบบมีแท็บใน `/admin`: ผู้ใช้ · แผนก · หมวดหมู่งาน · หมวดหมู่ทรัพย์สิน · สถานที่ · ระดับความสำคัญ · บทบาท/สิทธิ์ · **ตั้งค่า** · อีเมล · สำรองข้อมูล · บันทึกตรวจสอบ · ความปลอดภัย

---

## 1. ตั้งชื่อระบบ / โลโก้ / เวลาทำการ (แท็บ "ตั้งค่า")

`/admin` → แท็บ **ตั้งค่า** — แก้แล้วกดบันทึก มีผลทันที:

| ตั้งอะไรได้ | ใช้ทำอะไร |
|---|---|
| **ชื่อระบบ (app_name)** | ชื่อที่โชว์บนแถบข้าง อีเมล และหัวเอกสาร PDF |
| **สโลแกน (app_tagline)** | ข้อความรองใต้ชื่อ (แถบข้าง + อีเมล + หัว PDF) |
| **โลโก้ (logo)** | อัปโหลดโลโก้องค์กร (ขึ้นบนแถบข้าง + หัว PDF) · มีปุ่มลบโลโก้ด้วย |
| **เขตเวลา (default_timezone)** | เวลาที่ระบบใช้แสดง/คำนวณ (เช่น `Asia/Bangkok`) |
| **เวลาทำการ (business_start / business_end)** | ชั่วโมงทำงาน — ใช้ประกอบการคิด SLA |
| **คำนำหน้าเลขงาน (ticket_prefix)** | คำนำหน้ารหัสใบแจ้งซ่อม |

> 🎨 **อยากเปลี่ยนสีธีม** (คราม-ม่วง → สีองค์กร)? ทำได้โดยไม่ต้อง build — ดู [CUSTOMIZE.md](CUSTOMIZE.md)

## 2. จัดการผู้ใช้ (แท็บ "ผู้ใช้")

- **เพิ่มผู้ใช้:** กรอกชื่อผู้ใช้/อีเมล/ชื่อ-นามสกุล/รหัสผ่าน แล้วเลือก **บทบาท**
- **บทบาท (roles) มี 4 ระดับ:**
  - **ผู้แจ้ง (requester)** — แจ้งซ่อม ดูงานตัวเอง
  - **ช่าง (technician)** — รับงาน อัปเดตสถานะ บันทึกเวลา
  - **หัวหน้า (manager)** — มอบหมาย/ย้ายงาน ดูภาพรวม
  - **ผู้ดูแล (admin)** — ทำได้ทุกอย่าง รวมถึงตั้งค่าระบบ
- **ปิดการใช้งาน** ผู้ใช้ที่ลาออกได้ (ไม่ต้องลบ — เก็บประวัติงานไว้)
- **นำเข้าผู้ใช้จำนวนมาก:** มีปุ่มนำเข้าไฟล์ CSV (ดาวน์โหลดไฟล์แม่แบบได้จากหน้านั้น)
  > ⚠️ นำเข้าครั้งละไม่เกินหลักสิบรายการเพื่อกันหมดเวลา (แต่ละรายการเข้ารหัสรหัสผ่านซึ่งช้าโดยตั้งใจ)

## 3. โครงสร้างองค์กร (แผนก / หมวดหมู่ / สถานที่ / ความสำคัญ)

แต่ละแท็บเพิ่ม-แก้-ลบ รายการตั้งต้นได้:

- **แผนก (departments)** — หน่วยงานที่แจ้ง/รับผิดชอบงาน
- **หมวดหมู่งาน (categories)** — ประเภทงานซ่อม + **กำหนด SLA (ชั่วโมง) ต่อหมวดได้**
- **ระดับความสำคัญ (priorities)** — เช่น ปกติ/ด่วน/ด่วนมาก + เวลาตอบสนอง
- **หมวดหมู่ทรัพย์สิน (asset-categories)** + **สถานที่ (locations)** — สำหรับทะเบียนทรัพย์สิน/QR
  > 💡 **เลิกใช้ทรัพย์สิน:** ระบบไม่มีปุ่ม "ลบ" ทรัพย์สิน (เพื่อคงประวัติงานซ่อมที่อ้างถึงไว้) — ให้แก้ **สถานะ** เป็น "เลิกใช้งาน" หรือ "จำหน่าย" ในหน้าแก้ไขทรัพย์สินแทน (เหมือนการปิดใช้งานผู้ใช้)

## 4. อีเมล (แท็บ "อีเมล" + เมนูที่เกี่ยวข้อง)

- **ตั้งค่าการส่ง (SMTP):** ตั้งใน **ไฟล์ `.env`** (ค่า `MAIL_*`) — ไม่ได้ตั้งในหน้าเว็บ (ดู [INSTALL.md](INSTALL.md) ขั้นที่ 4)
- **ทดสอบส่งอีเมล:** แท็บ **อีเมล** → กรอกอีเมลปลายทาง → เลือกเทมเพลต → ส่งทดสอบ (ไม่โชว์รหัส SMTP)
- **แก้ข้อความอีเมล (เทมเพลต):** `/admin/email-templates` → เลือกเทมเพลต → แก้หัวข้อ/เนื้อหา → มีปุ่ม **รีเซ็ต** กลับค่าเดิมได้
- **ดูคิวอีเมล:** `/admin/email-queue` → เห็นอีเมลที่รอส่ง/ส่งแล้ว/ล้มเหลว + ปุ่ม **ลองส่งใหม่**
- **ประกาศถึงผู้ใช้ (broadcast):** `/admin/broadcast` → ส่งข้อความแจ้งถึงผู้ใช้ตามบทบาท

> อีเมลถูกส่งจริงโดย **cron** (ทุก 5 นาที) — ถ้าไม่ตั้ง cron อีเมลจะค้างในคิว (ดู INSTALL.md ขั้นที่ 6)

## 5. สำรอง & กู้คืนข้อมูล (แท็บ "สำรองข้อมูล")

- **ดูสถานะ:** เห็นว่าไฟล์สำรองล่าสุดเมื่อไหร่ / มีกี่ชุด / ค้างนานไหม
- **สำรองทันที:** ปุ่ม **"สำรอง & ดาวน์โหลด"** — สร้างไฟล์ `.sql.gz` ให้ดาวน์โหลดเก็บ
  **แนะนำให้กดก่อนอัปเดตระบบทุกครั้ง** (ทางถอยเวลามีปัญหา) · ทำงานได้แม้โฮสต์ปิดสิทธิ์พิเศษ
- **สำรองอัตโนมัติ:** ทำผ่าน cron รายวัน (ดู INSTALL.md ขั้นที่ 6) — เก็บย้อนหลังหลายชุดแล้วลบเก่าอัตโนมัติ
- **กู้คืน:** นำไฟล์ `.sql` (แตกจาก `.gz`) ไป **Import ผ่าน phpMyAdmin** เข้าฐานข้อมูล (หน้าจอมีคำสั่งกำกับ)

## 6. ตรวจสอบและความปลอดภัย

- **บันทึกตรวจสอบ (audit):** แท็บ **บันทึกตรวจสอบ** — ประวัติการกระทำสำคัญของผู้ดูแล (ใคร ทำอะไร เมื่อไหร่)
- **ความปลอดภัย (security):** แท็บ **ความปลอดภัย** — ประวัติความพยายามล็อกอิน (จับการเดารหัสถี่ ๆ)
- ระบบจำกัดการล็อกอินผิดถี่อัตโนมัติ (rate limit) และหมดเวลา session เมื่อไม่ใช้งานนาน

## 7. ข้อมูลตัวอย่าง (Demo) — สำหรับตอนทดลอง

- จะโหลดข้อมูลตัวอย่างได้ต้องตั้ง **`ALLOW_DEMO_DATA=true`** ใน `.env` ก่อน (บนของจริงให้เป็น `false`)
- โหลดผ่านปุ่มในหน้า admin (โผล่เฉพาะตอนยังไม่มีงานในระบบ) หรือติ๊กใน Setup Wizard
- **ล้างข้อมูลตัวอย่างเพื่อใช้จริง:** ให้ติดตั้งใหม่บนฐานข้อมูลเปล่า (Import `schema.sql` + `seed_reference.sql` โดยไม่โหลด demo) แล้วสร้าง admin ผ่าน wizard

## 8. สิ่งที่ผู้ใช้ทั่วไปทำเองได้ (ไม่ต้องพึ่ง admin)

- **ลืมรหัสผ่าน:** หน้า login → "ลืมรหัสผ่าน" → รับลิงก์รีเซ็ตทางอีเมล (ต้องตั้ง SMTP + cron)
- **แก้โปรไฟล์:** เมนูโปรไฟล์ → แก้ชื่อ/อีเมล/รหัสผ่านตัวเอง
- **ตั้งค่าการแจ้งเตือน:** โปรไฟล์ → การแจ้งเตือน → เลือกว่าจะรับแจ้งเตือนเรื่องไหน

> 📘 **การทำงานประจำวันของแต่ละบทบาท** (ผู้แจ้ง/ผู้จัดการ/ช่าง แจ้งซ่อม→อนุมัติ→มอบหมาย→ซ่อม→ปิดงาน)
> ดูใน [USER-GUIDE.md](USER-GUIDE.md) — คู่มือนี้เน้นการ *ตั้งค่าระบบ* ส่วนนั้นเน้นการ *ใช้งานประจำวัน*
