# 🎨 คู่มือปรับแต่ง (CUSTOMIZE) — สำหรับผู้ซื้อ Template

สรุปว่า **อะไรปรับเองได้ผ่านหน้าเว็บ/ไฟล์ตั้งค่า (ไม่ต้องแก้โค้ด)** และ **อะไรต้องให้ dev แก้**
ระบบออกแบบให้เรื่องที่ทุกองค์กรต้องเปลี่ยน (แบรนด์/สี/หมวดหมู่/อีเมล) ทำเองได้หมด

---

## ✅ ปรับเองได้ (ไม่ต้องแก้โค้ด)

| อยากเปลี่ยน | ทำที่ไหน | ต้องแก้โค้ดไหม |
|---|---|---|
| ชื่อระบบ / สโลแกน / โลโก้ | admin → แท็บ **ตั้งค่า** | ❌ ไม่ต้อง |
| **สีธีม** (คราม-ม่วง → สีองค์กร) | ไฟล์ `public/assets/css/theme.css` (ดูวิธีข้างล่าง) | ❌ ไม่ต้อง build |
| เขตเวลา / เวลาทำการ | admin → แท็บ **ตั้งค่า** | ❌ |
| คำนำหน้าเลขใบงาน | admin → แท็บ **ตั้งค่า** | ❌ |
| แผนก / หมวดหมู่งาน / ระดับความสำคัญ / สถานที่ | admin → แท็บที่เกี่ยวข้อง | ❌ |
| **SLA** (เวลาต่อหมวด/ความสำคัญ) | admin → แท็บ **หมวดหมู่งาน / ระดับความสำคัญ** | ❌ |
| ข้อความอีเมลแจ้งเตือน | admin → `/admin/email-templates` (แก้/รีเซ็ตได้) | ❌ |
| ผู้ส่ง/เซิร์ฟเวอร์อีเมล (SMTP) | ไฟล์ `.env` (ค่า `MAIL_*`) | ❌ (แก้ไฟล์ตั้งค่า) |
| จำนวนไฟล์แนบ/ขนาดไฟล์ที่อัปได้ | ไฟล์ `.env` (`UPLOAD_ATTACHMENT_MAX_FILES`, `UPLOAD_ATTACHMENT_MAX_BYTES`) | ❌ |
| จำนวนแถวสูงสุดในตารางรายงานทรัพย์สิน | ไฟล์ `.env` (`REPORT_ASSET_DISPLAY_LIMIT`) | ❌ |
| เปิด/ปิดข้อมูลตัวอย่าง | ไฟล์ `.env` (`ALLOW_DEMO_DATA`) | ❌ |

## 🎨 วิธีเปลี่ยนสีธีม (ไม่ต้อง build)

1. เปิดไฟล์ **`public/assets/css/theme.css`** ด้วยโปรแกรมแก้ข้อความ (Notepad / File Manager → Edit)
2. แก้ค่าเลขสี HEX ในบล็อก `:root` ให้เป็นสีขององค์กร เช่น:
   ```css
   --indigo-600: #059669;   /* สีปุ่มหลัก / ลิงก์ */
   --indigo-500: #10b981;   /* กรอบโฟกัส / ไฮไลต์ */
   --indigo-700: #047857;   /* ตัวอักษรเน้น */
   ```
3. บันทึก แล้ว **รีเฟรชหน้าเว็บ** — ปุ่ม/ลิงก์/เมนู/ป้ายจะเปลี่ยนสีทันที **ไม่ต้องรัน build ใด ๆ**

> 📌 **ข้อจำกัด:** แสงเรือง (glow) และเงาบางจุดฝังสีเดิมไว้ จึงไม่เปลี่ยนตาม — แต่ปุ่ม ลิงก์ ป้ายสถานะ
> ไล่เฉดหลัก และกรอบโฟกัส เปลี่ยนตามครบ (คำอธิบายเต็มอยู่ในหัวไฟล์ `theme.css`)

## ⚙️ ปรับได้แต่ต้องใช้เครื่องมือ dev เล็กน้อย

| อยากเปลี่ยน | ทำยังไง | หมายเหตุ |
|---|---|---|
| ดีไซน์/เลย์เอาต์เชิงโครงสร้าง (ระยะห่าง, การจัดวาง) | แก้ `resources/css/app.css` แล้วรัน `./build-css.sh` | สคริปต์โหลดตัว Tailwind ให้เอง (ไม่ต้องมี Node) — ผลออกที่ `public/assets/css/app.css` |
| ข้อความ/ป้ายในหน้าเว็บ | แก้ในไฟล์ `app/Views/…` | เป็นข้อความในเทมเพลต ต้องรู้จักโครงไฟล์เล็กน้อย |

## 🔴 ต้องให้ dev แก้ (ยอมรับว่าต้องจ้าง/เขียนโค้ด)

| อยากเปลี่ยน | ทำไมต้อง dev |
|---|---|
| ขั้นตอนสถานะงาน (workflow / state machine) | เป็นตรรกะธุรกิจในโค้ด (`app/Services/…`) |
| สิทธิ์แต่ละบทบาท (ใครทำอะไรได้) | ฝังในชั้นตรวจสิทธิ์ของโค้ด |
| **เพิ่มบทบาทใหม่** (เช่น "หัวหน้าช่าง") | ต้องแก้ `app/Support/Role.php` + เพิ่มค่าใน ENUM ของตาราง `users` |
| เพิ่มฟิลด์ใหม่ในใบแจ้งซ่อม/ทรัพย์สิน | ต้องแก้ทั้งฐานข้อมูล + ฟอร์ม + โค้ด |
| ผูกระบบภายนอก (LINE Notify, API อื่น) | ต้องเขียนโค้ดเชื่อมต่อใหม่ |
| เพิ่มรายงานประเภทใหม่ | ต้องเขียน service/หน้ารายงานใหม่ |

> 💡 **แนวคิดของ template นี้:** เรื่องที่ทุกองค์กรต้องปรับตอนเริ่มใช้ (แบรนด์ สี หมวดหมู่ SLA อีเมล) ทำเองได้หมด
> ส่วนที่ต้อง dev คือ "การเปลี่ยนพฤติกรรมของระบบ" ซึ่งเป็นเรื่องเฉพาะทางที่ควรมีคนเขียนโค้ดดูแล
