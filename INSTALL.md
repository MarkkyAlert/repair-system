# 📥 คู่มือติดตั้ง (INSTALL) — สำหรับ IT support

คู่มือนี้พาติดตั้งระบบบน **shared hosting / cPanel** ทีละขั้น **โดยไม่ต้องแก้โค้ด** ทำตามจากบนลงล่างได้เลย
ใช้เวลาประมาณ 20–40 นาที

> 🧭 **ศัพท์ที่จะเจอ:** `.env` = ไฟล์ตั้งค่า (ข้อความธรรมดา) · `phpMyAdmin` = เครื่องมือจัดการฐานข้อมูลใน cPanel · `cron` = ตัวตั้งเวลาให้ระบบทำงานเบื้องหลังอัตโนมัติ · `docroot` = โฟลเดอร์ที่เว็บเสิร์ฟ (มักชื่อ `public_html`)

---

## ✅ ก่อนเริ่ม — เช็กลิสต์ความพร้อม

ติ๊กให้ครบก่อนลงมือ:

- [ ] มีพื้นที่โฮสต์ที่ใช้ **PHP 8.1 ขึ้นไป** (ดูใน cPanel → *Select PHP Version*)
- [ ] มี **MySQL/MariaDB** และเข้า **phpMyAdmin** ได้
- [ ] เข้า **cPanel File Manager** หรือมีบัญชี **FTP** เพื่ออัปไฟล์
- [ ] ตั้ง **Cron Jobs** ได้ (cPanel → *Cron Jobs*)
- [ ] มีไฟล์แพ็กระบบ (`.zip`) ที่ได้รับจากผู้ขาย

> 💡 **วิธีเช็กเซิร์ฟเวอร์อัตโนมัติ:** หลังอัปไฟล์ (ขั้นที่ 1) ให้เปิด `https://your-site/check-requirements.php`
> หน้านั้นจะบอกเป็นภาษาไทยว่า PHP/ส่วนขยาย/ฐานข้อมูล/สิทธิ์เขียนไฟล์ **พร้อมหรือขาดอะไร** (✅/⛔) — ทำตามที่มันบอก

## 1. อัปโหลดไฟล์ระบบ

**ทางเลือก A — cPanel File Manager (แนะนำ):**
1. เข้า cPanel → **File Manager**
2. เข้าโฟลเดอร์ **`public_html`** (หรือโฟลเดอร์โดเมนของคุณ)
3. กด **Upload** → เลือกไฟล์ `.zip` ที่ได้รับ
4. กลับมาที่ File Manager → คลิกขวาที่ zip → **Extract** (แตกไฟล์)

**ทางเลือก B — FTP:** ใช้โปรแกรม FileZilla เชื่อมต่อ แล้วลากไฟล์ทั้งหมด (ที่แตก zip แล้ว) ขึ้นไปที่ `public_html`

> ℹ️ **ไม่ต้องรัน `composer`** — ไฟล์ไลบรารี (`vendor/`) แถมมาในแพ็กแล้ว
> (ถ้าคุณได้ไฟล์มาจาก Git โดยตรงและไม่มีโฟลเดอร์ `vendor/` → ต้องให้ dev รัน `composer install --no-dev` ให้ก่อน)

> 🔒 **ให้ปลอดภัยที่สุด (ถ้าทำได้):** ตั้ง docroot ของโดเมนให้ชี้ไปที่โฟลเดอร์ **`public/`** ข้างในระบบ
> (cPanel → *Domains* → เปลี่ยน Document Root) เพื่อให้ไฟล์ `.env`/โค้ดอยู่นอกพื้นที่เว็บ
> ถ้าเปลี่ยนไม่ได้ก็ไม่เป็นไร — ระบบมี `.htaccess` กันไฟล์ลับหลุดให้อยู่แล้ว

## 2. สร้างฐานข้อมูล (phpMyAdmin / MySQL Databases)

1. cPanel → **MySQL® Databases**
2. **Create New Database** → ตั้งชื่อ (เช่น `repair_system`) → กด Create
3. เลื่อนลง **Add New User** → ตั้ง username + รหัสผ่านที่คาดเดายาก → Create User
4. **Add User To Database** → เลือก user + database ที่เพิ่งสร้าง → ให้สิทธิ์ **ALL PRIVILEGES**
5. **📝 จดค่า 4 อย่างนี้ไว้** (ใช้ในขั้นที่ 4):

   | ค่า | ตัวอย่าง |
   |---|---|
   | ชื่อฐานข้อมูล (DB_NAME) | `cpaneluser_repair_system` |
   | ชื่อผู้ใช้ (DB_USERNAME) | `cpaneluser_repair` |
   | รหัสผ่าน (DB_PASSWORD) | (ที่คุณตั้ง) |
   | โฮสต์ (DB_HOST) | มักเป็น `localhost` หรือ `127.0.0.1` |

---

## 3. นำเข้าโครงสร้างฐานข้อมูล (Import .sql)

1. cPanel → **phpMyAdmin** → คลิกเลือกฐานข้อมูลที่สร้างไว้ (แถบซ้าย)
2. แท็บ **Import** → **Choose File** → เลือกไฟล์ **`database/schema.sql`** → กด **Go/Import**
   *(ไฟล์นี้สร้างตารางทั้งหมด — ใช้ตอนติดตั้งใหม่บนฐานข้อมูลเปล่าเท่านั้น)*
3. Import อีกครั้ง โดยเลือก **`database/seed_reference.sql`** (ข้อมูลตั้งต้น เช่น แผนก/หมวดหมู่/priority)
4. ✅ เสร็จแล้วควรเห็น **27 ตาราง** ในฐานข้อมูล และยัง **ไม่มีผู้ใช้** (จะสร้างในขั้น Setup Wizard)

> ⚠️ **อย่านำเข้า `schema.sql` ซ้ำบนระบบที่ใช้งานจริง** — มันลบข้อมูลเดิมทั้งหมด (ใช้เฉพาะตอนติดตั้งใหม่)
> จะอัปเกรดระบบเดิมให้ใช้ไฟล์ในโฟลเดอร์ `database/upgrades/` แทน

## 4. ตั้งค่าไฟล์ `.env`

1. ใน File Manager หา **`.env.production.example`** → คลิกขวา → **Copy** → เปลี่ยนชื่อสำเนาเป็น **`.env`**
   *(หรือ Rename `.env.production.example` เป็น `.env` ถ้าไม่ต้องเก็บต้นฉบับ)*
2. คลิกขวาที่ `.env` → **Edit** แล้วแก้ค่าเหล่านี้ (ค่าที่เหลือปล่อยตามค่าเริ่มต้นได้):

   ```ini
   APP_URL=https://your-real-domain.com     # URL จริง ไม่มี / ท้าย (สำคัญ! ใช้ทำลิงก์อีเมล/QR)
   DB_HOST=localhost                          # ค่าที่จดจากขั้นที่ 2
   DB_NAME=cpaneluser_repair_system
   DB_USERNAME=cpaneluser_repair
   DB_PASSWORD=รหัสที่ตั้งไว้
   ```
3. **ส่งอีเมลจริง** (ถ้าต้องการ) — ตั้งค่า SMTP ในไฟล์เดียวกัน (เอาค่าจากผู้ให้บริการอีเมลของคุณ):
   ```ini
   MAIL_DRIVER=smtp
   MAIL_HOST=smtp.your-provider.com
   MAIL_PORT=587
   MAIL_USERNAME=your-email@your-domain.com
   MAIL_PASSWORD=รหัสอีเมล
   MAIL_ENCRYPTION=tls
   MAIL_FROM_ADDRESS=noreply@your-domain.com
   ```
   *(ยังไม่พร้อมตั้ง SMTP? ปล่อย `MAIL_DRIVER=log` ไปก่อน ระบบจะไม่ส่งอีเมลจริงแต่ใช้งานส่วนอื่นได้)*
4. บันทึกไฟล์

> ℹ️ **ทุกค่ามีคำอธิบายภาษาไทย** อยู่ในไฟล์ `.env.example` เปิดดูประกอบได้

## 5. เปิด Setup Wizard (สร้างผู้ดูแลคนแรก)

1. เปิดเบราว์เซอร์ไปที่ `https://your-domain.com`
2. ระบบจะ **พาไปหน้า "ตั้งค่าระบบครั้งแรก" อัตโนมัติ** — กรอก:
   - **ชื่อระบบ** (เช่น "ศูนย์ซ่อมบำรุง บริษัท ABC") — แก้ทีหลังในหน้า admin ได้
   - **ผู้ดูแลคนแรก:** ชื่อผู้ใช้ (3–50 ตัว a–z 0–9 . - _), อีเมล, ชื่อ-นามสกุล, **รหัสผ่าน (อย่างน้อย 8 ตัว)**
   - *(ช่อง "โหลดข้อมูลตัวอย่าง" จะโผล่เฉพาะถ้าตั้ง `ALLOW_DEMO_DATA=true` — ตอนใช้งานจริงไม่ต้องติ๊ก)*
3. กดยืนยัน → ระบบพาไปหน้า **เข้าสู่ระบบ** → ล็อกอินด้วยบัญชีที่เพิ่งสร้าง → เข้าแดชบอร์ดได้เลย ✅

## 6. ⚠️ ตั้ง Cron (สำคัญมาก — ห้ามข้ามถ้าใช้ SLA/อีเมล)

**ถ้าไม่ตั้ง cron:** SLA จะไม่ถูกนับว่าเกินกำหนด และอีเมลแจ้งเตือนจะไม่ถูกส่ง — **แบบเงียบ ๆ ไม่มี error**
(ระบบไม่ได้พัง แค่ลืมตั้งตัวจับเวลา)

1. cPanel → **Cron Jobs**
2. Common Settings เลือก **"Once Per Five Minutes"** (`*/5 * * * *`)
3. ช่อง Command ใส่ (แทน `/path/to` ด้วยที่อยู่จริงของระบบบนโฮสต์):
   ```
   php /home/cpaneluser/public_html/bin/run-maintenance-cron.php
   ```
   > 💡 **ไม่รู้ path หรือคำสั่ง php ที่ถูกต้อง?** เปิด `https://your-site/check-requirements.php`
   > หน้านั้น **พิมพ์คำสั่ง cron ที่ตรงกับเครื่องคุณให้เลย** — ก็อปไปวางได้
4. (แนะนำ) เพิ่มอีกงานสำรองฐานข้อมูลรายวัน — เลือก "Once Per Day" แล้วใส่:
   ```
   php /home/cpaneluser/public_html/bin/backup-database.php
   ```
5. (ทางเลือก) เก็บกวาดไฟล์แนบกำพร้ารายสัปดาห์ — เลือก "Once Per Week" แล้วใส่:
   ```
   php /home/cpaneluser/public_html/bin/cleanup-orphan-attachments.php
   ```

## 7. ตรวจว่าติดตั้งสำเร็จ (Smoke Test) + เก็บกวาด

- [ ] ล็อกอินด้วยบัญชี admin ได้
- [ ] สร้างรายการแจ้งซ่อมทดสอบได้ 1 รายการ
- [ ] เปิดหน้า **รายงาน** ได้ (ยังไม่มีข้อมูลก็ไม่เป็นไร)
- [ ] (ถ้าตั้ง SMTP) ทดสอบส่งอีเมล: admin → แท็บ **อีเมล** → ส่งอีเมลทดสอบ
- [ ] **ลบไฟล์ `public/check-requirements.php`** ออกจากเซิร์ฟเวอร์ (เพื่อความปลอดภัย — มันเป็นแค่เครื่องมือตอนติดตั้ง)
- [ ] ตรวจว่า `APP_ENV=production` และ `APP_DEBUG=false` ใน `.env`

🎉 **เสร็จสมบูรณ์** — ต่อไปดู [ADMIN-GUIDE.md](ADMIN-GUIDE.md) เพื่อตั้งค่าองค์กร (โลโก้/แผนก/ผู้ใช้)

## 🔧 แก้ปัญหาที่พบบ่อย (Troubleshooting)

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ (IT support ทำเองได้) |
|---|---|---|
| หน้าขาว / **Error 500** ตอนเปิดเว็บ | PHP เวอร์ชันต่ำไป / `.env` ผิด / สิทธิ์ไฟล์ | เปิด `check-requirements.php` เช็กก่อน; ตั้ง PHP ≥ 8.1 ใน cPanel; ตรวจว่ามีไฟล์ `.env` และค่าถูกต้อง |
| ขึ้น **"ระบบเริ่มทำงานไม่สำเร็จ (รหัสอ้างอิง…)"** | ต่อฐานข้อมูลไม่ได้ / ยังไม่ import schema | เปิด `check-requirements.php` — จะบอกว่า DB ต่อได้ไหม/ตารางครบไหม แล้วแก้ตามนั้น |
| เปิดเว็บแล้ว **ไม่เด้งไป /setup / หน้า error** | ยังไม่ได้ import `schema.sql` | กลับไปทำขั้นที่ 3 (Import) ให้ครบ |
| **เชื่อมฐานข้อมูลไม่ได้** | ค่า `DB_*` ใน `.env` ไม่ตรงกับที่สร้างใน phpMyAdmin | ตรวจ DB_HOST/DB_NAME/DB_USERNAME/DB_PASSWORD ให้ตรงกับที่จดไว้ (cPanel มักเติม prefix ชื่อบัญชีให้อัตโนมัติ) |
| **ตัวอักษรไทยเพี้ยน** (□□□) | charset ฐานข้อมูลไม่ใช่ utf8mb4 / import ผิด encoding | สร้างฐานข้อมูลใหม่เป็น `utf8mb4`; ตอน Import ใน phpMyAdmin เลือก charset `utf8mb4` |
| **อัปโหลดรูปแนบไม่ได้** | โฟลเดอร์ `storage/` เขียนไม่ได้ | ใน File Manager คลิกขวาโฟลเดอร์ `storage` → **Change Permissions** → ตั้ง `755` (หรือ `775`) ให้เขียนได้ |
| **อีเมลไม่ส่ง** | ตั้ง `MAIL_DRIVER=log` / ค่า SMTP ผิด / ไม่ได้ตั้ง cron | ตั้ง `MAIL_DRIVER=smtp` + ค่า SMTP ให้ถูก; ตั้ง cron (ขั้นที่ 6); ทดสอบที่ admin → แท็บอีเมล |
| **SLA ไม่อัปเดต / ไม่มีแจ้งเตือน** | ยังไม่ได้ตั้ง cron | ทำขั้นที่ 6 (cron ทุก 5 นาที) |
| **รายงานไม่มีข้อมูล** | ยังไม่มีงานพอ / cron ยังไม่รัน | ปกติ — ข้อมูลจะขึ้นเมื่อมีการแจ้งซ่อม/ปิดงานจริง |
| เปิด **`/setup` ไม่ได้ (เด้งไป login)** | ติดตั้งเสร็จแล้ว (ปกติ) | ถูกต้องแล้ว — Setup Wizard ทำได้ครั้งเดียวตอนยังไม่มี admin |
| เปิด **`.env` หรือไฟล์ใน `config/` ผ่าน URL แล้ว 403** | `.htaccess` กันไว้ตั้งใจ (ปลอดภัย) | เป็นเรื่องดี — ไฟล์ลับต้องเปิดผ่าน File Manager ไม่ใช่ผ่านเว็บ |
| ปุ่ม/หน้ากด **"สำรอง & ดาวน์โหลด" ช้า/ไฟล์ใหญ่** | ข้อมูลเยอะ (ปกติ) | รอสักครู่ ระบบสร้างไฟล์ `.sql.gz` ให้ดาวน์โหลด — ควรกดก่อนอัปเดตระบบทุกครั้ง |

> ❓ แก้ตามตารางแล้วยังไม่หาย ให้จด **"รหัสอ้างอิง"** ที่หน้าจอแสดง แล้วส่งให้ผู้ขาย — รหัสนี้ผูกกับ log บนเซิร์ฟเวอร์
