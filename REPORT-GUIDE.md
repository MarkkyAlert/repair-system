# 📊 คู่มืออ่านรายงาน (REPORT-GUIDE) — สำหรับผู้บริหาร

ระบบมีรายงาน 9 แบบ ในเมนู **รายงาน (`/reports`)** ส่งออกเป็น **CSV / Excel / PDF** ได้ (รองรับภาษาไทย)
คู่มือนี้ช่วยให้ผู้บริหาร **อ่านตัวเลขให้ถูก** และรู้ว่าแต่ละรายงานตอบคำถามอะไร

> 📖 **นิยาม/สูตร/เกณฑ์สีที่เป๊ะที่สุด อยู่ในระบบเอง:** เมนู **รายงาน → คู่มือ (`/reports/guide`)**
> หน้านั้นอธิบาย metric + เกณฑ์ + ข้อควรระวังทุกตัว และ **ผูกกับสูตรจริงในโค้ด** (อัปเดตพร้อมกันเสมอ)
> เอกสารนี้จึงเป็น "ภาพรวมสำหรับผู้บริหาร" ไม่เขียนสูตรซ้ำเพื่อกันตัวเลขไม่ตรงกัน

---

## รายงานแต่ละแบบตอบคำถามอะไร

| รายงาน | ตอบคำถามผู้บริหาร | ดูทิศทางไหนว่า "ดี" |
|---|---|---|
| **ภาพรวมผู้บริหาร** (executive) | สรุปสุขภาพงานซ่อมทั้งหมดในหน้าเดียว | การ์ดสรุปเป็นสีเขียว = ดี |
| **SLA เกินกำหนด** (sla-breach) | เราทำงานทันเวลาที่สัญญาไว้แค่ไหน | % ตรงเวลา **สูง** = ดี / จำนวนเกินกำหนด **ต่ำ** = ดี |
| **ผลงานช่าง** (technician-performance) | ช่างแต่ละคนรับ/ปิดงานเท่าไร เร็วแค่ไหน | ปิดงานได้มาก + เวลาเฉลี่ยต่ำ = ดี |
| **จุดเสียบ่อย** (problem-hotspot) | อะไร/ที่ไหนเสียซ้ำ ๆ (ควรแก้ที่ต้นเหตุ) | จุดที่ตัวเลขสูง = ควรเข้าไปจัดการ |
| **แนวโน้ม** (trend) | ปริมาณงานเพิ่ม/ลดตามเวลา | ดูทิศทางเส้นกราฟประกอบบริบท |
| **งานค้าง** (backlog-aging) | มีงานค้างนานแค่ไหน เก่าสุดกี่วัน | งานค้างน้อย + ไม่เก่า = ดี |
| **เปิดซ้ำ** (reopen-rate) | งานที่ปิดแล้วถูกเปิดใหม่บ่อยไหม (คุณภาพงาน) | % เปิดซ้ำ **ต่ำ** = ดี |
| **ความพึงพอใจ** (csat) | ผู้แจ้งให้คะแนนงานซ่อมยังไง | คะแนนเฉลี่ย **สูง** = ดี |
| **ความน่าเชื่อถือทรัพย์สิน** (asset-reliability) | เครื่อง/ทรัพย์สินไหนเสียบ่อย/ดาวน์ไทม์สูง | ดาวน์ไทม์/จำนวนเสีย **ต่ำ** = ดี |

## 3 เรื่องที่ผู้บริหารต้องรู้ก่อนตีความ

1. **ข้อมูลน้อย อย่าเพิ่งสรุป** — ถ้ามีงานไม่กี่รายการ เปอร์เซ็นต์จะแกว่งง่าย (เช่น ปิดงาน 1 จาก 2 = 50%)
   ระบบจะ **แสดงขนาดฐานข้อมูล (เช่น "จาก N งาน")** กำกับไว้ — ให้ดูฐานประกอบทุกครั้ง
2. **ช่องว่าง (– หรือ —) ไม่เท่ากับศูนย์** — หมายถึง "ยังไม่มีข้อมูลพอจะคำนวณ" ไม่ใช่ผลงานแย่
3. **ตัวเลขสดต้องมี cron** — SLA/สถานะบางอย่างอัปเดตโดยงานเบื้องหลัง ถ้าไม่ได้ตั้ง cron ตัวเลขอาจไม่เป็นปัจจุบัน
   (ดู [INSTALL.md](INSTALL.md) ขั้นที่ 6)

## การส่งออกและนำไปใช้

- ทุกรายงาน **ส่งออก CSV / Excel / PDF** ได้ (ปุ่มบนหน้ารายงาน) — PDF/Excel รองรับภาษาไทย เหมาะแนบรายงานประชุม
- มี **ชุดตัวอย่างรายงาน** (`/reports/sample-pack`) สำหรับดูหน้าตาผลลัพธ์
