# 👥 คู่มือการใช้งานประจำวัน (USER-GUIDE) — แยกตามบทบาท

คู่มือนี้อธิบาย **การทำงานประจำวันของแต่ละบทบาท** ในระบบแจ้งซ่อม — ผู้ใช้ทุกคนอ่านเฉพาะ
ส่วนของบทบาทตัวเองได้เลย (การตั้งค่าระบบสำหรับ admin อยู่ใน [ADMIN-GUIDE.md](ADMIN-GUIDE.md))

ระบบมี 4 บทบาท: **ผู้แจ้ง (requester)** · **ผู้จัดการ (manager)** · **ช่าง (technician)** · **ผู้ดูแล (admin)**
และมีช่องทางแจ้งแบบ **ไม่ต้องล็อกอินผ่าน QR (guest)** สำหรับคนนอก (ดูส่วน "แจ้งผ่าน QR" ด้านล่าง)

---

## 🔄 ภาพรวมเส้นทางของงานซ่อม (ใครทำอะไร เมื่อไหร่)

งานซ่อมหนึ่งใบเดินผ่านมือแต่ละบทบาทตามลำดับนี้:

```
[ผู้แจ้ง] แจ้งซ่อม
      ↓  (รออนุมัติ)
[ผู้จัดการ] อนุมัติ / ไม่อนุมัติ
      ↓  (อนุมัติแล้ว)
[ผู้จัดการ] มอบหมายช่าง
      ↓  (มอบหมายแล้ว)
[ช่าง] รับงาน → เริ่มงาน → สรุปผลการซ่อม
      ↓  (รอตรวจรับ — รอผู้แจ้งยืนยัน)
[ผู้แจ้ง] ยืนยันปิดงาน + ให้คะแนน   ── หรือ ──  ขอแก้งานซ้ำ (ถ้ายังไม่เรียบร้อย)
      ↓
   ปิดงาน ✅
```

> 💡 **สถานะงาน** ที่เห็นในระบบ: รออนุมัติ → อนุมัติแล้ว → มอบหมายแล้ว → รับงานแล้ว →
> กำลังดำเนินการ → รอตรวจรับ → เสร็จสิ้น (หรือ: ถูกปฏิเสธ / ยกเลิกแล้ว)

## 🙋 ผู้แจ้ง (Requester) — พนักงานทั่วไปที่แจ้งของเสีย

**สิ่งที่ทำประจำวัน:**

### 1. แจ้งซ่อม
- เมนู **"แจ้งซ่อม"** → กรอกฟอร์ม: หัวข้อปัญหา, รายละเอียด, หมวดหมู่งาน, สถานที่, ระดับความสำคัญ
- แนบรูปได้ (เช่น รูปของที่เสีย) — ช่วยให้ช่างเข้าใจปัญหาเร็วขึ้น
- กดส่ง → งานเข้าสถานะ **"รออนุมัติ"** รอผู้จัดการอนุมัติ

### 2. ติดตามสถานะงานของตัวเอง
- เมนู **"งานของฉัน"** → เห็นงานทั้งหมดที่เคยแจ้ง + สถานะปัจจุบันของแต่ละงาน
- กดเข้าไปดูรายละเอียด: ใครรับผิดชอบ, ความคืบหน้า, หมายเหตุจากช่าง

### 3. ยืนยันปิดงาน + ให้คะแนน (สำคัญ!)
- เมื่อช่างซ่อมเสร็จ งานจะเป็นสถานะ **"รอตรวจรับ"** และคุณจะได้แจ้งเตือน
- เข้าไปที่งานนั้น → ตรวจว่าซ่อมเรียบร้อยจริงไหม:
  - **เรียบร้อยแล้ว** → กด **"ยืนยันปิดงาน"** + **ให้คะแนนความพึงพอใจ 1–5 ดาว**
  - **ยังไม่เรียบร้อย** → กด **"ขอแก้งานซ้ำ"** + ระบุเหตุผล → งานกลับไปหาช่างอีกครั้ง

> ⚠️ **งานจะไม่ปิดเองอัตโนมัติ** — ต้องผู้แจ้งยืนยันเท่านั้น ถ้าไม่ยืนยัน งานจะค้างที่ "รอตรวจรับ"
> (คะแนนที่ให้จะไปโผล่ในรายงานความพึงพอใจของผู้บริหาร)

### 4. อื่น ๆ
- **ยกเลิกงาน** ได้ก่อนมอบหมายช่าง (ตอนสถานะ "รออนุมัติ" หรือ "อนุมัติแล้ว") — ถ้าแจ้งผิด/ไม่ต้องการแล้ว
- **ปิดงานไปแล้วแต่กลับมามีปัญหาอีก** → ใช้ปุ่ม **"ทำซ้ำ"** เปิดใบงานใหม่จากรายการเดิม (งานที่ "เสร็จสิ้น" แล้วจะกด "ขอแก้งานซ้ำ" ไม่ได้ เพราะถือว่าจบแล้ว)

## 🔓 แจ้งผ่าน QR โดยไม่ต้องล็อกอิน (Guest) — สำหรับคนนอก/คนที่ไม่มีบัญชี

ทุกทรัพย์สิน/เครื่องมีสติกเกอร์ **QR Code** ติดไว้ ใครเจอของเสียก็แจ้งได้ทันที **โดยไม่ต้องมีบัญชี**
(เหมาะกับแม่บ้าน ผู้มาติดต่อ หรือพนักงานที่ยังไม่มี user)

1. **สแกน QR** บนเครื่องด้วยกล้องมือถือ → เปิดหน้าที่แสดง **ข้อมูลเครื่องนั้น** (ชื่อ / รหัส / ที่ตั้ง) ให้แน่ใจว่าแจ้งถูกเครื่อง
2. กด **"แจ้งปัญหา"** → กรอกฟอร์มสั้น ๆ (ไม่ต้องล็อกอิน): **หัวข้อ + รายละเอียดปัญหา** และ **ชื่อ / อีเมล / เบอร์โทร** ของผู้แจ้ง (ไว้ติดต่อกลับ)
3. กดส่ง → คำแจ้งเข้าระบบเป็น **"คำขอจากภายนอก"** สถานะ *รอตรวจ* — ยังไม่ใช่ใบงาน จนกว่าแอดมิน/ผู้จัดการจะรับเรื่อง
4. **ติดตามสถานะเอง** → หน้ายืนยันหลังกดส่งจะให้ **เลขอ้างอิง (ref)** พร้อมปุ่ม **"ติดตามสถานะคำขอ"** → เก็บเลขไว้ แล้วเข้าหน้า **ติดตามสถานะ (`/track`)** กรอกเลข ref + เบอร์/อีเมลที่แจ้งไว้ ดูความคืบหน้าได้เองโดยไม่ต้องล็อกอิน

**ฝั่งแอดมิน/ผู้จัดการรับเรื่อง** — เมนู **"คำขอจากภายนอก" (`/admin/guest-requests`)**:
- **แปลงเป็นใบงาน** → เลือกหมวดหมู่ + ระดับความสำคัญ → กลายเป็นงานซ่อมปกติ เดินเข้าเส้นทางอนุมัติ→มอบหมายตามเดิม
- **ปฏิเสธ** → ถ้าเป็นสแปมหรือไม่เกี่ยวข้อง

> 💡 ระบบกันสแปม/กดซ้ำอัตโนมัติ (จำกัดจำนวนครั้งต่อ IP + กันกดส่งซ้ำด้วย token ครั้งเดียว) — ถ้าขึ้น "คำขอนี้ถูกส่งไปแล้ว" ให้สแกน QR ใหม่
> 💡 ถ้าคนที่สแกนเป็นพนักงานที่ **ล็อกอินอยู่** ระบบจะพาไปฟอร์มแจ้งซ่อมปกติ (เลือกเครื่องให้อัตโนมัติ) แทนฟอร์ม guest

## 👔 ผู้จัดการ (Manager) — อนุมัติงานและมอบหมายช่าง

**สิ่งที่ทำประจำวัน:**

### 1. อนุมัติงานที่รอ
- เมนู **"รออนุมัติ"** → เห็นงานใหม่ที่ผู้แจ้งส่งเข้ามา
- กดเข้าดูรายละเอียด → ตัดสินใจ:
  - **อนุมัติ** → งานพร้อมมอบหมายช่าง
  - **ไม่อนุมัติ** → ระบุเหตุผล → แจ้งกลับผู้แจ้ง
- **อนุมัติหลายงานพร้อมกัน** ได้ (เลือกหลายรายการ → อนุมัติทีเดียว) สำหรับงานที่เข้ามาเยอะ

### 2. มอบหมายช่าง
- งานที่อนุมัติแล้ว → กด **"มอบหมายช่าง"** → เลือกช่างที่ว่าง/เหมาะกับงาน
- ระบบจะแจ้งเตือนช่างคนนั้นอัตโนมัติ (ถ้าตั้ง SMTP + cron)

### 3. ติดตามภาพรวม
- แดชบอร์ด → เห็นงานทั้งหมด สถานะ SLA (ใกล้เกินกำหนด/เกินแล้ว), งานค้าง, ภาระงานช่างแต่ละคน
- ใช้ตัดสินใจมอบหมายงานให้สมดุล (ไม่ให้ช่างคนใดคนหนึ่งงานล้น)

> 💡 **ผู้จัดการเห็นรายงานผู้บริหารได้** — SLA, ผลงานทีมช่าง, พื้นที่ปัญหา, งานเปิดซ้ำ
> (ดูวิธีอ่านใน [REPORT-GUIDE.md](REPORT-GUIDE.md))

## 🔧 ช่าง (Technician) — รับงานและซ่อม

**สิ่งที่ทำประจำวัน:**

### 1. ดูงานที่ได้รับมอบหมาย
- เมนู **"งานของฉัน"** → เห็นงานที่ผู้จัดการมอบหมายให้ + กำหนดเวลา (SLA)
- เรียงตามความสำคัญ/กำหนดเวลา — งานด่วนขึ้นก่อน

### 2. เดินงานทีละขั้น
- **รับงาน** (accept) → ยืนยันว่ารับทราบและจะทำงานนี้
- **เริ่มงาน** (start) → ตอนลงมือซ่อมจริง (ระบบเริ่มจับเวลาดำเนินการ)
- **สรุปผลการซ่อม** (resolve) → เมื่อซ่อมเสร็จ: กรอกสิ่งที่วินิจฉัย + วิธีแก้ + อะไหล่ที่ใช้ (ถ้ามี)
  → งานเปลี่ยนเป็น **"รอตรวจรับ"** รอผู้แจ้งยืนยัน

> 💡 **บันทึกให้ละเอียด** ตอนสรุปผล — ข้อมูลนี้ไปโผล่ในประวัติทรัพย์สินและรายงาน
> (เช่น ของชิ้นนี้ซ่อมบ่อยแค่ไหน = รายงานสุขภาพทรัพย์สิน)

### 3. ถ้าถูกขอแก้งานซ้ำ
- ถ้าผู้แจ้งกด "ขอแก้งานซ้ำ" งานจะกลับมาที่คุณพร้อมเหตุผล → ซ่อมใหม่แล้วสรุปผลอีกครั้ง
- (งานเปิดซ้ำถูกนับในรายงาน — ช่วยให้ทีมเห็นว่างานไหนแก้ไม่จบในครั้งเดียว)

## 🛡️ ผู้ดูแล (Admin) — ดูแลระบบ + ทำได้ทุกอย่าง

Admin ทำงานของทุกบทบาทข้างบนได้ **และ** ดูแลระบบ (ตั้งค่า/ผู้ใช้/สำรองข้อมูล) —
รายละเอียดการตั้งค่าทั้งหมดอยู่ใน [ADMIN-GUIDE.md](ADMIN-GUIDE.md)

**งานประจำวันของ admin (นอกจากการตั้งค่า):**
- ดูภาพรวมทั้งระบบผ่านแดชบอร์ด + รายงานผู้บริหารทุกตัว
- จัดการเมื่อมีปัญหา: รีเซ็ตรหัสผ่านผู้ใช้, แก้บทบาท, ดูบันทึกตรวจสอบ (audit log) ว่าใครทำอะไร
- **รับเรื่อง "คำขอจากภายนอก" (guest)** ที่คนนอกแจ้งผ่าน QR → แปลงเป็นใบงานหรือปฏิเสธ (ดูส่วน "แจ้งผ่าน QR")
- สำรองข้อมูลก่อนอัปเดต/เปลี่ยนแปลงใหญ่

## ❓ คำถามที่ผู้ใช้ถามบ่อย

| คำถาม | คำตอบ |
|---|---|
| แจ้งซ่อมแล้วทำไมยังไม่มีคนทำ? | รอผู้จัดการอนุมัติ + มอบหมายช่างก่อน ดูสถานะที่ "งานของฉัน" |
| งานซ่อมเสร็จแล้วแต่ยังไม่ปิด? | ผู้แจ้งต้องกด "ยืนยันปิดงาน" เอง ระบบไม่ปิดอัตโนมัติ |
| ช่างซ่อมไม่เรียบร้อย ทำไง? | กด "ขอแก้งานซ้ำ" + ระบุเหตุผล งานจะกลับไปหาช่าง |
| ไม่มีบัญชี แจ้งซ่อมได้ไหม? | ได้ — สแกน QR ที่ติดบนเครื่อง → กรอกฟอร์มแจ้ง (ไม่ต้องล็อกอิน) admin จะรับเรื่องแปลงเป็นใบงาน |
| ลืมรหัสผ่าน? | หน้า login → "ลืมรหัสผ่าน" → รับลิงก์ทางอีเมล (ต้องตั้ง SMTP + cron) |
| ไม่ได้รับอีเมลแจ้งเตือน? | admin ต้องตั้ง SMTP + cron ให้ก่อน (ดู INSTALL ขั้น 6) |
| อยากเปลี่ยนเรื่องที่รับแจ้งเตือน? | เมนูโปรไฟล์ → การแจ้งเตือน → เลือกเอง |

## หมายเหตุ

- ⚠️ **ชื่อเมนู/ปุ่มที่เห็นจริง** อาจต่างจากคู่มือเล็กน้อยตามการตั้งค่าของแต่ละองค์กร — ยึดตามที่เห็นในระบบจริง
- 🧪 **แนะนำ:** ให้ผู้ใช้จริงของแต่ละบทบาทลองทำงานตามคู่มือ 1 รอบ เพื่อยืนยันว่าเข้าใจตรงกัน
